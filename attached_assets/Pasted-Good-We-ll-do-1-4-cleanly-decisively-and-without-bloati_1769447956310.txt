1️⃣ GAME-READINESS INDEX — EXACT SCORING RULES

This is not a vibe score. It’s a controlled judgment based on things you already compute.

Scale

0–100, where:

0–30 = Not game-ready

31–60 = Limited game readiness

61–80 = Competitive readiness

81–100 = Advanced game readiness

No “elite.” No league names.

Inputs you already have

classification

diagnoses

timing_gap_frames

pitch_type

head drift (boolean)

Scoring logic (deterministic, transparent)

Start at 100, subtract penalties.

Core penalties

Early commit → −30

Arm-dominant swing → −25

Head drift / balance loss → −20

Large timing gap (>10 frames) → −15

Simultaneous start (no separation) → −10

Pitch-context penalties

Fastball vulnerability → −10

Breaking-ball vulnerability → −10

Floor + ceiling

Minimum score: 0

Maximum score: 100

That’s it. Simple. Explainable. Defensible.

Early commit + head drift + fastball vulnerability:

100
-30 early commit
-20 head drift
-10 fastball vulnerability
= 40 → Limited game readiness


That feels right to coaches.

DROP-IN CODE (Python)

Add this function:

def calculate_game_readiness(classification, diagnoses, timing_gap, pitch_issues):
    score = 100

    if classification == "early_commit":
        score -= 30
    if classification == "arm_dominant_swing":
        score -= 25

    if "head_drift_balance_loss" in diagnoses:
        score -= 20

    if timing_gap and timing_gap > 10:
        score -= 15

    if classification == "simultaneous_start":
        score -= 10

    for issue in pitch_issues:
        score -= 10

    return max(0, min(100, score))
